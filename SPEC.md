## 5. JTSI — 저널리즘 전환 스트레스 지수 (핵심 수식)

### 5.1 정의
> **JTSI** (Journalism Transformation Stress Index) = 디지털 전환이 개별 언론인에게 가하는 누적 스트레스를 하나의 수치로 표현한 자체 지수.

### 5.2 구성 (4요소)
| 요소 | 변수 | 의미 |
|---|---|---|
| 디지털 피로 | `digital_fatigue` | 디지털 대응·혁신 피로감 |
| AI 리스크 인식 | `ai_risk` | AI가 유발하는 위험 인식 |
| 역할 실행 갭 | `role_gap` | 이상과 현실의 괴리 |
| 번아웃 | `burnout` | 탈진·직업 회의 |

### 5.3 수식 (★ 반드시 z-score 표준화 후 합산)
```
JTSI_i = z(피로_i) + z(AI리스크_i) + z(역할갭_i) + z(번아웃_i)

여기서 z(x_i) = (x_i − mean(x)) / std(x)
```
**왜 z-score인가**: 4요소의 척도가 다름(피로 1~5, 역할갭 -4~+4 등). 단순 합산하면 분산 큰 변수가 지수를 지배. 표준화로 각 요소를 동등 가중.

```python
from scipy.stats import zscore
comp = ['digital_fatigue','ai_risk','role_gap','burnout']
dd = df.dropna(subset=comp).copy()
for c in comp:
    dd[c+'_z'] = zscore(dd[c])
dd['JTSI'] = dd[[c+'_z' for c in comp]].sum(axis=1)
```

### 5.4 검증값 (2025, n=2020)
- JTSI 범위: -8.50 ~ +8.70, 평균 ≈ 0 (z합산 특성)
- AI 활용자 평균 +0.029 / 비활용자 -0.040
- 매체별: 뉴스통신 +0.245 > 방송 +0.120 > 신문 +0.063 > 인터넷 -0.441

해석: 인터넷언론사가 스트레스 가장 낮고, 전통매체(통신·방송)가 높음 → 정책 타겟 시사.

### STEP 5.5 — JTSI 민감도 검토
**목적**: "이 지수 임의로 만든 것 아니냐"는 의심에 대응. 구성을 바꿔 결론이 *얼마나* 안정적인지 정직하게 점검한다(전부 안정적이라 단정하지 않음 — 실제로 단순모델 v2에선 순위가 흔들림).
**방법**: 3개 버전을 계산하고 상위 위험집단 순위를 비교.
| 버전 | 구성 | 성격 |
|---|---|---|
| JTSI v1 (기본) | 피로 + AI리스크 + 역할갭 + 번아웃 | 메인 |
| JTSI v2 (단순) | 피로 + AI리스크 | 최소 |
| JTSI v3 (확장) | v1 + 직무불만족(10−만족도) | 확장 |
**판단 기준**: 세 버전에서 고위험 매체·연령 집단의 순위 상관(Spearman ρ)이 높으면(>0.7) "안정적". 순위가 뒤집히면 메인 버전 재검토.
산출물: `outputs/tables/jtsi_sensitivity.csv`
**보고서 활용(보수적)**: "인터넷언론사의 상대적 저위험은 지수 구성과 무관하게 일관되게 나타났다(세 버전 모두 최저). 다만 전통매체 간 세부 순위는 구성에 따라 변동하므로(v1-v2 순위상관 0.40), 개별 매체 순위보다 '전통매체 대 인터넷매체'라는 큰 구도로 해석한다."

### STEP 6 — 통계적 타당성 확인 (★진단보다 먼저)
**목적**: STEP 4·6.5에서 "집단 간 차이가 있다"고 주장하려면, 그 차이가 우연이 아님을 먼저 검정으로 확인.
**방법**:
- 두 집단 평균 차이(활용자 vs 비활용자) → Welch 독립표본 t검정(equal_var=False) `scipy.stats.ttest_ind`
- 3집단 이상 평균 차이(매체·연령대별 JTSI) → 일원분산분석 `scipy.stats.f_oneway`
- 범주형 연관(매체 × AI활용여부 등) → 카이제곱 `scipy.stats.chi2_contingency`

**효과크기 보고 규칙**:
- t-test → Cohen's d
- ANOVA → eta squared
- 카이제곱 → Cramér's V

p값은 차이가 우연일 가능성을 판단하는 기준이고, 효과크기는 차이의 실제 크기를 판단하는 기준이다. 효과크기가 작은 경우에는 통계적으로 유의하더라도 '강한 차이'로 표현하지 않는다.
**판단 기준**: p<0.05면 "유의한 차이". p≥0.05면 단정 금지 — "경향이 관찰됨"으로만 서술.
**유의성 표기**: p<0.05 *, p<0.01 **, p<0.001 ***
산출물: `outputs/tables/significance_tests.csv`

### STEP 6.5 — 위기집단 진단 (2×2 분석틀)
**목적**: 검정으로 확인된 차이를 바탕으로, 정책 타겟이 될 고위험 집단을 명확히 규정.
**군집분석 검토 결과**: 실데이터 실루엣 점수가 0.16~0.20 수준으로 낮아 자연스러운 군집 구조가 뚜렷하지 않았다. 따라서 본 분석에서는 해석 가능성과 정책 연결성이 높은 2×2 분석틀을 사용한다.
**대신 명시적 2×2 분석틀 사용:**
```
               디지털 피로 낮음   디지털 피로 높음
AI 활용함     ① 안정 적응형      ② AI 과부하형 ★타겟
AI 활용 안함   ③ 전환 관망형         ④ 전환 소외형 ★타겟
```
- 분할 기준: 피로는 **4점 이상/미만**(4='대체로', 5='매우 그렇다'=피로 호소), AI는 q38(2025).
- 각 사분면의 JTSI·만족도·번아웃 평균과 표본수(n) 보고.
- ②AI 과부하형, ④전환 소외형이 핵심 정책 대상.
산출물: `notebooks/06_diagnosis.ipynb`

## 8. 분석 재현 시 유의사항

1. **CSV 쓰지 말 것** — 2025 CSV 손상. SAV만 사용.
2. **변수명 직접 타이핑 금지** — 반드시 variable_mapping 경유. q5/q6 혼동 주의.
3. **JTSI는 z-score 후 합산** — 단순합산 금지.
4. **AI 리스크 연도비교는 공통 2문항만** — 2025 오보문항(q39_5)은 2023에 없음.
5. **가중치 없음** — 집단 비교 시 표본수(n) 항상 병기.
6. **저작권** — 원본 데이터 깃허브 업로드 금지. 보고서 인용은 출처 명기.
7. **단순 아이디어 나열 금지** — 모든 제안은 분석 수치를 근거로(공모 규칙).
